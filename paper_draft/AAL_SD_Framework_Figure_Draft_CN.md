# AAL-SD 方法框架图草案

本文件用于给 IEEE JSTARS 论文中的主方法框架图提供可直接绘制的中文说明。

## 推荐图题

所提出 AAL-SD 框架的整体架构

## 图的目标

展示完整的主动学习闭环，并明确说明 AAL-SD 不只是一个打分规则，而是一个包含以下组成部分的闭环系统：

- 模型训练与评估
- 不确定性与特征提取
- AD-KUCS 打分
- LLM-agent 辅助选样
- 基于标注反馈、检查点和风险监控的闭环更新

## 推荐布局

建议使用“主闭环 + 侧向监控分支”的流程图结构，使审稿人可以一眼识别：方法是闭环系统而非单次打分器。

### 版本 A（单图）

从左到右绘制主流程，并用一条回路箭头返回训练模块；同时从训练模块引出侧向监控分支，回流到 `λ_t` 控制与 Agent。

### 版本 B（双子图，推荐用于避免拥挤）

- 子图 (a)：主动学习主循环（只保留 6 个主模块 + 1 条回路）
- 子图 (b)：控制与安全机制（监控信号 → `λ_t` 控制器与 Agent 工具箱）

### 主流程

1. **数据集划分**
   - 初始标注池 `L0`
   - 未标注池 `U`
   - 固定测试集 `T`

2. **语义分割模型训练**
   - DeepLabV3+
   - 在当前标注池 `Lt` 上训练
   - 在 `T` 上评估

3. **未标注池推理**
   - 像素级概率图
   - 不确定性分数 `U(x)`
   - 深层特征表示 `f_x`

4. **AD-KUCS 打分模块**
   - 不确定性分支
   - 知识增益分支
   - 动态 lambda 控制器
   - 融合得分：`Score(x) = (1 - λ_t)U(x) + λ_tK(x)`

5. **LLM Agent 决策模块**
   - 系统提示词 + 工具箱
   - 获取系统状态
   - 获取 top-k 候选样本
   - 最终提交选择
   - 输出可解释理由

6. **标注与样本池更新**
   - 查询子集 `Qt`
   - 人工/专家标注
   - 更新 `Lt -> Lt+1`

7. **进入下一轮**
   - 用箭头返回训练模块

### 上方监控分支

从训练模块引出一个侧向分支，用于表示：

- mIoU 轨迹
- F1 轨迹
- 梯度对齐 / 过拟合风险
- 回滚阈值与检查点管理器

该分支应同时输入到：

- 动态 lambda 控制器
- LLM agent 工具箱

### 避免“看起来很乱”的绘图约束（强烈建议遵守）

- 主流程每个模块最多 3 行文字，避免在模块内塞入长公式/长列表
- 只保留两类回流箭头：`标注更新 -> 训练` 与 `监控 -> (λ_t, Agent)`
- 工具箱与信号列表放在子图 (b) 或者侧边小框，不要占用主流程块内空间
- 颜色分组固定：训练/推理（蓝）、AD-KUCS（橙）、Agent（绿）、监控/安全（灰或红）

## 推荐图注

图1. AAL-SD 的整体架构。在每一轮主动学习中，当前语义分割模型首先在标注池 `Lt` 上训练，并在固定测试集 `T` 上评估；随后对未标注池 `U` 进行推理，获得不确定性估计 `U(x)` 和特征表示 `f_x`。AD-KUCS 通过自适应权重因子 $\lambda_t$ 融合不确定性与知识增益分数 `K(x)`，形成候选排序。LLM agent 通过工具箱接口读取系统状态、候选排序与安全信号，并据此确定查询子集 `Qt`，同时输出可解释的选择理由。新标注样本回流至标注池并更新未标注池；检查点与风险信号被记录并用于下一轮控制与决策。

## 绘图建议

- 训练/推理主路径建议使用蓝色。
- AD-KUCS 打分模块建议使用橙色。
- LLM-agent 模块建议使用绿色。
- 监控/安全模块建议使用灰色或红色。
- 监控分支应在视觉上与主流程适度分离，使审稿人能够一眼看出该方法具有闭环控制特征。
- 若主流程仍显拥挤，优先采用双子图版本，而不是继续向模块内增加文本。
