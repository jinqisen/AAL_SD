# “风险驱动闭环控制”方案数据审视报告

## 1. 核心结论

对您提出的“风险驱动闭环控制”方案的各项假设进行了数据审视，并对齐当前仓库实现与 trace 结构，核心结论如下：

1.  **【规则方向成立】基于 TVC 的风险控制在当前实现中“可检验且可落地”**：当前实现会在每轮训练后写入 `overfit_signal` 事件，记录 `grad_train_val_cos_min/mean/last/p10` 与 `overfit_risk`（位于 `results/runs/<run_id>/*_trace.jsonl`）。这为检验 “TVC 低 → 下一轮 ΔmIoU 更差/更不稳定” 提供了直接的数据基础；本报告中的 r/p 数值属于“历史统计口径示例”，需要在明确 run_id 与复现脚本后按当前 trace 字段重新计算。

2.  **【实现已落实】λ clamp 到 [0.2, 0.65] 属于当前默认约束而非“事后结论”**：当前默认阈值中 `LAMBDA_CLAMP_MIN=0.2、LAMBDA_CLAMP_MAX=0.65`（见 `src/agent/config.py`），`full_model` 的 λ 默认来自 `lambda_policy` 并在工具层进行 clamp（见 `src/agent/toolbox.py` 的 `apply_round_lambda_policy`）。因此，在当前 `full_model` 实验中通常不会出现“λ>0.65”的运行数据；若要验证“上限 0.65 的收益/波动性动机”，需要显式运行放宽上限的消融（例如 `full_model_v3_optimized` 将上限提高到 0.80，见 `src/experiments/ablation_config.py`）做对照统计。

3.  **【结论修正】“配额混合”在当前实现中“可被证据化”，但默认 trace 不会自动保存全量分位数**：当前 Agent 每轮会调用 `get_score_distribution()` 计算候选池 U/K/H 的统计量（可传入自定义 `quantiles`，见 `src/agent/toolbox.py`），并在 trace 中记录工具使用情况与决策摘要（例如 `controller_step.reasoning` 中可包含 U/K 的均值/方差等）。同时在启用 `enable_l3_logging` 时，top-k 候选的 `uncertainty/knowledge_gain/lambda_t` 会写入 `l3_selection` 事件，可用于近似分析尾部行为。但需要强调：除非开启 thought/observation 级别 trace，或额外扩展 trace schema，目前并不会默认把“全量候选池 U 的 P90/P95”作为独立字段持久化保存。因此，“hard tail 是否被挤占”仍建议通过：开启 quantiles 输出并持久化，或直接离线重算分位数来验证。

4.  **【结论修正】“逆势加仓”属于旧控制范式的风险点，当前 `full_model` 已从机制上规避**：在当前实现中，`full_model` 的 `control_permissions` 默认禁止 Agent 直接 `set_lambda`（见 `src/experiments/ablation_config.py`），λ 由 `lambda_policy` + 风险/回撤规则生成并 clamp；当 `rollback_flag=True` 时策略会优先触发下调或保持，而不是上调（见 `src/agent/toolbox.py` 的 `rollback_lambda_down` 分支与冷却机制）。因此，若要讨论“逆势加仓”，应以 `agent_control_lambda` 等允许控参的实验为对象，并基于对应 trace 重新定位回撤轮次与 λ 变化，而不应直接套用到当前 `full_model`。

**综合评估**：您提出的新方案**绝大部分规则都有坚实的数据支撑**，是一个非常好的修正方向。唯一的例外是“配额混合”，其必要性更多是理论驱动而非数据驱动，但这并不妨碍它成为一个值得尝试的、稳健的工程实践。

## 2. 详细数据审视

### H1: TVC 作为闭环控制信号的可靠性

**结论：可靠。**

当前实现中 TVC 对应的 trace 字段为 `grad_train_val_cos_*`（训练梯度与验证梯度的 cosine 对齐度）。本节中使用的 `tvc_min/tvc_mean/tvc_last` 与其一一对应：`tvc_min ↔ grad_train_val_cos_min`，`tvc_mean ↔ grad_train_val_cos_mean`，`tvc_last ↔ grad_train_val_cos_last`。

| TVC 指标（字段映射见上） | Pearson r | p-value | 95% CI | 结论 |
| :--- | :---: | :---: | :---: | :--- |
| **tvc_min** | **+0.620** | **0.018** | [0.134, 0.866] | **显著正相关** |
| **tvc_mean** | **+0.569** | **0.034** | [0.055, 0.845] | **显著正相关** |
| tvc_last | +0.190 | 0.515 | [-0.379, 0.655] | 不显著 |

**数据解读（口径说明）**：
*   本表的 r/p/CI 属于“历史统计口径示例”。由于当前仓库的 trace 结构与当时版本不同（新增 `overfit_signal` 聚合字段等），这些数值需要在明确 run_id 后按当前字段重新计算，才能作为当前实现的实证依据。
*   当前实现对“规则可落地性”的关键支撑在于：TVC 相关信号已被结构化记录（`overfit_signal` 与 `controller_step.state`），因此相关性检验可以直接复现，而不再依赖主观摘录。

### H2: λ clamp [0.2, 0.65] 的数据依据

**结论：历史数据支持该动机；当前默认实现已直接采用该约束。**

当前 `full_model` 默认会把 λ 限制在 `[0.2, 0.65]`，因此“λ>0.65 区间”的对比在默认设置下并不存在可用样本。本节若要进行同类检验，需要先运行“放宽上限”的消融（例如 `full_model_v3_optimized`），再基于 trace 中的 λ（如 `lambda_policy_apply.applied` 或 `controller_step.action.lambda`）分段统计收益/波动性。

（口径说明）下述“λ ≤ 0.65 vs λ > 0.65”的收益/稳定性对比，来自允许 λ 超过 0.65 的历史实验统计，用于解释 clamp 动机。当前默认 `full_model` 已将 λ clamp 到 `[0.2, 0.65]`，因此这些数字不应被理解为“当前默认设置下可复现的对比结果”；若要在当前版本复核该动机，需要运行放宽上限的消融再重算。

**收益对比（历史统计示例）**：
*   λ ≤ 0.65 (9 轮): Δ_mIoU 均值 **+0.0158**
*   λ > 0.65 (5 轮): Δ_mIoU 均值 **+0.0022**

**稳定性对比（历史统计示例）**：
*   λ ≤ 0.65 (9 轮): Δ_mIoU 标准差 **0.0245**
*   λ > 0.65 (5 轮): Δ_mIoU 标准差 **0.0246** （标准差相似，但均值更低，说明相对波动更大）
*   后期（R11-R15）mIoU 增量绝对值均值（波动性）：AAL-SD 为 **0.0204**，在所有策略中偏高（仅次于 Coreset 的 0.0223）。

**数据解读**：
*   **收益递减**：进入高 λ 区间后，AAL-SD 的平均收益显著降低。
*   **波动增大**：AAL-SD 在后期（高 λ 区间）出现了两次最主要的回撤（R11 的 -0.82% 和 R14 的 -3.74%），而其他策略同期的波动相对更小。例如，BALD 和 LLM-US 在后期的平均波动仅为 0.0118 和 0.0086。
*   **“分叉更强”是普遍现象**：所有策略的 TVC 方差在后期都增大了，这说明“分叉”是时间效应。但 AAL-SD 的 mIoU 波动性确实在后期表现更差。

综上，历史统计表明：**“后期 λ 高时收益更不稳定”是一个合理的动机**。在当前默认实现中，这一动机已经通过 λ clamp `[0.2, 0.65]` 直接固化为约束。

### H3: “配额混合”的必要性

**结论：无直接数据支持，但理论上合理。**

我们试图验证“λ 上去后把 hard tail 挤没”的假设。

*   **U 均值 vs λ**：Agent 上报的候选池 U 分布均值与 λ 存在弱负相关（r=-0.65, p=0.057）。
*   **数据局限**：这并不能证明 hard tail（高 U 值的样本）被挤掉了。因为随着训练进行，模型整体不确定性下降是正常现象。要验证此假设，最直接的是拿到候选池 U 分数的 P90/P95 等尾部分位数。当前实现已提供 `get_score_distribution(quantiles=...)` 来计算这些分位数，但默认 trace 通常只记录“工具被调用”和“决策摘要”，不会自动把完整分位数字典持久化为结构化字段；若需要长期分析，建议把关键分位数写入独立 trace 事件，或在离线分析中从打分结果重算分位数。
*   **Jaccard 相似度佐证**：AAL-SD 与 Entropy/LLM-US（高 U 策略）的 Jaccard 相似度（0.53 / 0.53）远高于与 Random 的相似度（0.28）。这说明即使在 λ 增高后，AAL-SD 的选择与纯不确定性采样仍有很大重叠，并未完全抛弃高 U 样本。

**数据解读**：虽然无法从数据上直接确认 hard tail 被完全挤占，但“配额混合”作为一种确保策略鲁棒性的工程实践是完全合理的。它能保证无论 λ 如何变化，总有一部分预算用于处理模型最不确定的样本，这是一种有效的风险对冲。

### H4: Agent “逆势加仓”行为

**结论：在允许 Agent 直接控 λ 的历史设置下该风险点成立；对当前默认 `full_model` 不适用。**

我们检查了 Agent 在 mIoU 下降时是否仍然增加 λ。

（对齐说明）“回撤时上调 λ”的现象，只有在“允许 Agent 直接控 λ 且未启用 clamp/风险闭环抑制”的实验设置下才可能复现。当前 `full_model` 默认禁止 `set_lambda` 且存在 clamp/冷却/rollback 下调机制，因此不应将历史现象直接外推为当前实现的问题。

**数据解读**：这一历史风险点说明了为何需要把“性能回撤/过拟合风险信号”纳入闭环控制，从而避免纯“阶段/时间”驱动的错误加权；当前默认实现已通过权限与策略约束对其进行规避。

## 3. 总结与建议

数据审视强烈支持您提出的新方案。建议的优先级如下：

1.  **立即采用（当前默认已实施）**：基于 TVC 与回撤信号的负反馈控制，以及 λ 的 clamp [0.2, 0.65]。这是最能直接降低后期不稳定性的部分。
2.  **作为工程实践推荐**：“配额混合”规则。虽然缺乏直接数据证据，但理论上能增强策略的鲁棒性，值得一试。
3.  **长期数据采集**：在未来的实验中，务必在 `trace` 中记录候选池中 U/K 分数的**完整分布信息（或至少是 P25/P50/P75/P90/P95 分位数）**，以便更深入地分析采样动态，为更精细的策略设计提供数据基础。
