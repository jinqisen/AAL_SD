## 二、EIL-NLAL框架核心架构

EIL-NLAL框架的核心是一个**交互式闭环**，它将领域专家、LLM Agent和主动学习算法紧密结合。其架构图如下：

```mermaid
graph TD
    subgraph 
    subgraph "EIL-NLAL 框架"
        A[领域专家] -->|自然语言指令| B(Prompt Engineer);
        B -->|优化Prompt| C(LLM Agent);
        C -->|决策 (样本选择)| D(AD-KUCS算法);
        D -->|更新权重λ_t| E(主动学习循环);
        E -->|模型性能/样本信息| C;
        E -->|反馈 (可视化/报告)| A;
        C -->|工具调用 (Tool Use)| F(工具箱: U(x), K(x), Model.predict);
    end
```

### 2.1 核心组件

1.  **领域专家 (Domain Expert)**：地质遥感专家，通过自然语言输入其领域知识、偏好或对Agent行为的调整指令。
2.  **Prompt Engineer (提示词工程师)**：负责将专家的自然语言指令转化为Agent可理解和执行的结构化Prompt。这可能涉及Prompt模板的动态调整、指令解析和冲突解决。
3.  **LLM Agent (大型语言模型智能体)**：
    *   **感知 (Perception)**：接收当前主动学习轮次的状态信息（模型性能、已标注样本特征、未标注样本的U(x)和K(x)值）。
    *   **思考 (Thought)**：基于接收到的信息、内置的AD-KUCS策略以及专家的自然语言指令，进行推理和决策。
    *   **行动 (Action)**：调用工具箱中的工具（如`calculate_uncertainty`, `calculate_knowledge_gain`, `select_samples`等）来执行样本选择。
    *   **学习 (Learning)**：通过专家的反馈，调整其内部策略或Prompt Engineer的转换逻辑。
4.  **AD-KUCS算法 (Adaptive Knowledge and Uncertainty-driven Sampling)**：主动学习的核心采样算法，其动态权重λ_t将受到Agent决策的影响。
5.  **主动学习循环 (Active Learning Loop)**：标准的AL循环，包括模型训练、样本标注、性能评估等。
6.  **工具箱 (Toolbox)**：Agent可调用的函数集合，用于获取数据信息、计算U(x)和K(x)、执行样本选择等。

### 2.2 “专家经验-提示词-算法权重”闭环

1.  **专家经验输入**：专家观察模型性能和Agent的决策，通过自然语言向Prompt Engineer提供反馈或指令。例如：“我认为模型对河流附近的滑坡识别效果不好，请Agent在下次选择样本时，多关注河流区域的样本。”
2.  **提示词转换**：Prompt Engineer将专家指令转化为Agent的Prompt。例如，将上述指令转化为：“在选择样本时，请特别关注那些靠近河流且模型不确定性较高的区域。这可能意味着这些区域包含模型尚未学习到的关键特征。”
3.  **Agent决策**：Agent接收包含专家指令的Prompt，结合AD-KUCS算法的U(x)和K(x)信息，进行样本选择。例如，Agent可能会调整其内部对λ_t的偏好，或者在选择样本时，优先考虑满足“靠近河流”条件的样本。
4.  **算法权重调整**：Agent的决策（例如，对特定区域或特征的偏好）会通过Prompt Engineer或直接通过Agent的工具调用，影响AD-KUCS算法中λ_t的动态调整，甚至可能引入新的权重因子。
5.  **模型性能提升**：通过专家引导的样本选择，模型在特定区域或整体性能上得到提升。
6.  **反馈与迭代**：提升后的模型性能和Agent的决策日志再次反馈给专家，形成新的迭代循环。

## 三、实验设计与评估方案

### 3.1 实验目标

1.  **验证EIL-NLAL的有效性**：证明专家通过自然语言干预能显著提升主动学习的效率和最终性能。
2.  **量化专家知识的价值**：评估不同专家干预策略对模型性能的影响。
3.  **评估可解释性与控制力**：证明专家对Agent决策的理解和控制能力。

### 3.2 数据集

*   **Landslide4Sense**：作为主要数据集，进行详细的对比和消融实验。
*   **LoveDA / GID**：作为辅助数据集，验证框架的泛化能力。

### 3.3 对比基线

1.  **传统主动学习方法**：Random, Entropy, Core-Set, BALD。
2.  **LLM Agent主动学习方法**：
    *   **AAL-SD (基础版)**：无专家干预的Agent。
    *   **AAL-SD + Fixed Prompt**：Agent使用固定Prompt，不接受专家干预。
3.  **人类专家独立标注**：作为理想上限，评估EIL-NLAL与纯人类标注的差距。

### 3.4 实验设置

*   **迭代轮数**：15-20轮。
*   **随机种子**：5个。
*   **专家干预频率**：每3-5轮进行一次专家干预。
*   **专家干预类型**：
    *   **通用策略调整**：例如，“请Agent在初期多关注不确定性高的样本。”
    *   **特定区域/特征关注**：例如，“请Agent关注河流附近的滑坡样本。”
    *   **错误纠正**：例如，“Agent上次选择的样本中，有很多是假阳性，请Agent下次更谨慎。”

### 3.5 评估指标

1.  **定量指标**：
    *   **ALC (Area Under Learning Curve)**：衡量学习效率。
    *   **mIoU (Mean Intersection over Union)**：衡量最终分割性能。
    *   **F1-score, Precision, Recall**：详细评估模型性能。
    *   **专家干预效率**：达到相同性能所需的专家干预次数/时间。
2.  **定性指标**：
    *   **专家满意度问卷**：评估专家对Agent决策的理解度、控制力、信任度。
    *   **Agent决策日志分析**：分析Agent如何响应专家指令，以及其决策逻辑的变化。

### 3.6 可视化方案

1.  **学习曲线对比图**：展示EIL-NLAL与所有基线的ALC和mIoU。
2.  **专家干预效果图**：在学习曲线上标注专家干预点，并展示干预前后的性能变化。
3.  **Agent决策分析图**：类似论文1，但重点展示专家指令如何影响Agent的Thought和Action。
4.  **专家知识对齐图**：可视化专家指令与Agent实际决策之间的相关性。
