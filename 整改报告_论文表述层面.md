## 1. 结论摘要（给论文直接可用的表述）

### 1.1 λ 轨迹“平台期=0.2”是否是实现/记录不一致？

结论：在默认 full-model 配置下，“闭环阶段 λ 在 0.2 附近出现较长平台期”**并非记录错误的必然证据**，而是与现有 λ-policy 的逻辑一致：warmup 固定为 0.2，进入风险闭环后默认规则是 `hold`，只有在触发 rollback / severe overfit / 低风险且低增益等条件时才会上下调，因此在信号不触发时会稳定停留在 0.2。

- 默认 full-model 配置（warmup_lambda=0.2，risk_control_start_round=4，delta_up/down 等）：[ablation_config.py](file:///Users/anykong/AD-KUCS/AAL_SD/src/experiments/ablation_config.py#L1-L120)

### 1.2 Agent 是否“真正读取图像内容”？

结论：当前实现中，LLM/Agent **不接收原始图像像素、patch 或任何可视化内容**。Agent 能访问的信息主要是：
- 全局训练/闭环指标（mIoU、rollback_flag、TVC/overfit_risk 等），通过 `get_system_status` 暴露给 LLM；
- 样本级仅提供“统计量/元信息”（例如 `image_mean`、`image_std`、`mask_positive_ratio`、image_name、是否在 labeled/unlabeled pool），通过 `get_sample_details` 暴露给 LLM；
- 选样分数（U/K/Hybrid）与分布统计（若调用相关工具）。

关键代码依据：

## 2. 需要在论文中明确补齐/修正的表述点

### 2.1 λ-policy（warmup + 风险闭环）应明确写出“会长期 hold”

建议在方法章节用“阶段式策略 + 触发条件”写清楚（避免读者误以为每轮都会动态变化）：

- **阶段 0：uncertainty-only（前 `uncertainty_only_rounds` 轮）**  
  λ 固定为 `r1_lambda`（默认可为 0.0），用于早期仅依赖不确定性查询。
- **阶段 1：warmup（从 warmup_start_round 起持续 warmup_rounds 轮）**  
  λ 固定为 `warmup_lambda`（full-model 默认 0.2）。若配置了 `warmup_lambda_range`，则按种子与轮次生成确定性均匀采样值（实现中使用 SHA256 哈希，不依赖 run_id/experiment_name）。
- **阶段 2：风险闭环（round ≥ risk_control_start_round）**  
  默认规则为 `hold`；仅在以下情形改变 λ：  
  - rollback_flag=True → λ 下调 delta_down（并可带冷却）  
  - severe overfit（由 overfit_risk 与 TVC 组合判定，支持 OR/AND、CI 触发）→ λ 下调  
  - 低风险 + 低增益（或 K 主导）→ λ 上调 delta_up  
  同时允许 EMA 平滑与最大步长约束，并最终 clamp 到 [LAMBDA_CLAMP_MIN, LAMBDA_CLAMP_MAX]。

对应实现见：[_compute_policy_lambda_for_round](file:///Users/anykong/AD-KUCS/AAL_SD/src/agent/toolbox.py#L123-L338)

### 2.2 图表/文本中对“闭环阶段”的定义要与实现一致

目前实现里的“闭环信号”来自每轮训练结束后的 overfit_signal 与 training_state（包含 rollback_flag / miou_delta / tvc 等），然后供 λ-policy 使用。建议论文明确：
- 闭环信号的来源：训练 epoch 内梯度对齐度（train-val cosine）聚合得到 TVC；再由 TVC 派生 overfit_risk。  
  代码：overfit_risk 计算与 trace 记录：[main.py](file:///Users/anykong/AD-KUCS/AAL_SD/src/main.py#L1325-L1370)
- rollback_flag 的定义（miou_delta 与阈值比较，阈值可能是固定或自适应）：[main.py](file:///Users/anykong/AD-KUCS/AAL_SD/src/main.py#L1460-L1500)

### 2.3 必须在论文中更正 Agent 的“视觉访问”表述

若论文现有表述暗示 LLM 直接“看图”或“理解图像内容”，需要改为更准确的实现描述（两种可选方向）：
