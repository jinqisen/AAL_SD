# 测试集闭环泄漏整改方案（可落盘）

## 1. 目的与结论口径

本整改的目标是：**确保任何会影响主动学习闭环决策的信号（λ 控制、回滚、epochs/预算控制、策略选择等）都仅来自训练集内部的验证/校准集合**，并将测试集仅用于最终一次性报告，从而避免 test-set leakage。

整改完成后，论文与实验报告口径需要同步调整：

- 主动学习过程中的曲线、回滚与控制信号：来自 **Validation/Calibration（V）**。
- 论文最终对外性能：来自 **Test（T）**（严格不参与任何闭环决策）。
- 若数据条件无法提供带标注的独立测试集，则必须把当前“test_pool”明确改称为验证集，并在论文中声明“无独立测试集，仅验证集评估”，同时弱化泛化结论。

## 2. 当前实现中的泄漏链路（证据点）

当前代码在训练循环中使用 `test_loader` 同时承担：

1) 每个 epoch 的评估指标（mIoU/F1），并把这些指标衍生成 `rollback_flag`/`miou_delta` 等闭环状态；  
2) 作为梯度对齐探针（`grad_probe_loader`）计算 `train_val_cos`，进一步衍生 `overfit_risk`，进入 λ 控制闭环。

关键位置：

- 数据池加载了 `test_pool.csv` 并映射为 `self.test_indices`：`src/main.py` 中加载 pools（`test_df` / `test_indices`）。
- 每轮训练时构造 `test_loader`（默认 `EVAL_SPLIT=internal` 时使用 `self.test_indices`），并在每个 epoch 调用 `trainer.evaluate(test_loader)`。
- 当 `GRAD_LOG_VAL_ALIGNMENT=True` 时，`trainer.train_one_epoch(..., grad_probe_loader=test_loader)`，即用 `test_loader` 做梯度探针。
- 训练结束后用探针梯度统计构造 `overfit_risk`，并将 `overfit_risk`/`rollback_flag` 等写入 `training_state`，通过 `Toolbox.set_training_state` 进入 λ 控制闭环。

这些链路只要 `test_loader` 对应的是论文对外报告的测试集 T，就构成 test-set leakage。

## 3. 整改原则（必须满足）

### P1：测试集只用于“冻结后评估”

“冻结后评估”指：模型参数、采样决策、控制参数（λ/回滚/epochs 等）已不再变化时，对测试集做一次性评估；评估结果不得再被写入任何会影响后续决策的状态容器（如 `training_state` / 控制器历史）。

### P2：闭环信号仅来自训练内的 V

闭环信号包括但不限于：

- 回滚/风险：`miou_delta`、`rollback_flag`、`overfit_risk`、梯度对齐统计等；
- 控制策略输入：`Toolbox.set_training_state` 的所有字段；
- Agent 可观测信息：`get_training_status` / `get_system_status` 对外暴露的性能指标；
- 任何 early stopping / best checkpoint 选择逻辑（若存在）。

### P3：命名与口径一致

代码/日志/论文中的 `test_*`、`val_*`、`eval_*` 必须语义一致，避免将“验证集当测试集写进论文”。

## 4. 推荐整改路径（两条可落盘路线）

### 路线 A（强推荐）：引入 V（验证/校准）与 T（严格测试）

#### 4.1 数据切分与池文件

目标分解：

- `L0`：初始标注集（已有）
- `U`：未标注池（已有）
- `V`：验证/校准集（新增或从现有“test_pool”转名）
- `T`：严格测试集（新增，**永不参与闭环**）

落盘方式（建议）：

1. 将当前 `test_pool.csv` 改为 `val_pool.csv`（或 `calib_pool.csv`），语义改为 V。  
2. 新增 `test_pool.csv`（严格测试 T），由数据预处理阶段一次性划分产生，并固定写入。

注意：若官方数据集自带 val/test 且具备像素级标注，可将 `V=官方val`、`T=官方test`；内部 pools 仅用于 L0/U 的主动学习过程，不再包含 test_pool。

#### 4.2 训练与闭环的 loader 责任划分

必须形成明确职责：

- `train_loader`：仅用于训练（来自 L0 的标注子集）。
- `val_loader`：用于每 epoch 评估与闭环信号（mIoU/F1、rollback、梯度探针、overfit_risk）。
- `test_loader`：仅在“冻结后评估”阶段使用；默认不在训练循环内调用；若必须生成学习曲线，使用 val 曲线替代。

#### 4.3 代码改动点（实施任务清单，暂不修改）

涉及文件与改动类型（仅列出需要做什么，不提供直接 patch）：

- `src/main.py`
  - 将当前 `test_loader` 一分为二：`val_loader` 与 `test_loader`。
  - 训练循环中：
    - `trainer.evaluate(...)` 改为评估 `val_loader`；
    - `grad_probe_loader` 改为 `val_loader`；
    - `epoch_mious`、`best_miou`、`rollback_flag`、`training_state` 的构造全部改基于 val 指标；
  - 测试评估：
    - 迁移到每轮结束后“冻结评估”步骤，或迁移到训练完成后的独立分析脚本中，严禁写回 `training_state`。

- `src/core/trainer.py`
  - 保持 `grad_probe_loader` 机制不变，但其上游必须只传入 `val_loader`。

- `src/agent/toolbox.py`
  - 无需改动策略逻辑本身，但需要保证其输入 `training_state` 不再包含 test 产生的信号。

#### 4.4 配置整改（建议新增/调整）

为防止未来回归，建议引入显式配置，而不是复用 `EVAL_SPLIT` 的“test/val 混用”语义：

- `CLOSED_LOOP_SPLIT`: `internal_val | official_val`（闭环与曲线使用的 split）
- `TEST_SPLIT`: `internal_test | official_test | disabled`（最终报告用）
- `NO_TEST_DURING_TRAINING`: `true/false`（科研护栏，默认 true）
- `GRAD_PROBE_SPLIT`: 与 `CLOSED_LOOP_SPLIT` 绑定，避免误传

### 路线 B（次优）：无法提供独立测试集时的口径修正

当数据条件无法提供带标注的严格测试集（T）时：

1. 把现有 `test_pool.csv` 明确更名为 `val_pool.csv`；  
2. 论文中把 “固定测试集 T” 改为 “固定验证集 V”；  
3. 仅报告验证集曲线与结果，不再声称测试集性能；  
4. 在 Related Work / Limitations 明确：缺少独立测试集，泛化结论待后续跨区域/跨数据集验证补充。

该路线能消除“测试集泄漏”指控，但会削弱“对外泛化性能”的证据力度。

## 5. 验收标准（必须可自动化检查）

### A. 静态验收（代码级）

满足以下任一方式即可判定通过：

1) 训练主循环中不再出现 `trainer.evaluate(test_loader)`；  
2) `grad_probe_loader` 不允许绑定到 `test_loader`；  
3) `training_state` 的构造字段（`last_miou/prev_miou/miou_delta/rollback_flag/overfit_risk/grad_train_val_cos_*`）明确来自 `val_loader` 计算链路。

建议新增一条“科研护栏”断言：当 `NO_TEST_DURING_TRAINING=true` 时，一旦在训练循环内触发 test loader 的 evaluate/probe，立即抛错并落 trace/status。

### B. 行为验收（运行级）

在开启闭环控制的实验中：

- 训练日志与 trace 中闭环信号（λ、rollback、overfit_risk）与 val 指标同源；
- test 指标仅出现在“冻结后评估”阶段的独立段落/独立脚本输出；
- 相同 seed 下，关闭 test 过程评估不应改变主动学习选样结果（因为闭环不再读取 test 信号）。

### C. 论文口径验收（报告级）

- 所有 “test” 字样必须对应“严格冻结后评估”的集合；
- 学习曲线、回滚与 λ 轨迹说明应明确是 val/calibration；
- 若使用路线 B，必须删除或弱化任何“测试集性能”结论。

## 6. 回归风险与缓解

- 风险：val 指标波动更大导致控制器更敏感。  
  - 缓解：为闭环信号引入平滑（EMA）与窗口分位阈值（已有风险策略框架可复用），并在论文中透明报告。

- 风险：历史结果与新口径不可直接对比。  
  - 缓解：将旧结果标注为“含 test-in-loop 的历史版本”，新结果用新的 run_id 与 manifest 隔离，并在图表脚注说明。

## 7. 实施顺序（建议）

1. 先落盘数据池语义（V/T 划分与文件命名），确保团队统一口径；  
2. 再修改训练主循环的 loader 责任；  
3. 最后补齐护栏配置与静态/运行验收用例；  
4. 重新跑 4 seeds 的主实验与关键消融，更新论文图表与方法叙述。
